# GitHub Actions Dashboard

## Project Structure

- `src/GitHubActionsDashboard.Api/` — .NET backend (ASP.NET Core minimal API)
- `src/GitHubActionsDashboard.App/` — React frontend (Vite + TanStack Router)

## Build Commands

### Backend
```bash
cd src/GitHubActionsDashboard.Api && dotnet build
```

### Frontend
```bash
cd src/GitHubActionsDashboard.App && npm run build
```

## API Client Generation

The frontend API client (`src/GitHubActionsDashboard.App/src/api/`) is auto-generated from the backend's OpenAPI spec. **Do not manually edit files in `src/api/`.**

To regenerate after backend model/endpoint changes:

1. Start the backend (it must be running on http://localhost:5010):
   ```bash
   cd src/GitHubActionsDashboard.Api && dotnet run --launch-profile http
   ```
2. In a separate shell, regenerate the client:
   ```bash
   cd src/GitHubActionsDashboard.App && npm run generate
   ```
3. Stop the backend process.

The `http` launch profile runs on port 5010 without HTTPS, which matches the openapi-ts config.

## Key Conventions

- Backend handlers are static classes with a `Handle` method; routes are mapped in `Program.cs`
- Backend services extend `GitHubService` base class (retry logic, caching)
- Frontend uses TanStack Router file-based routing — the route tree (`routeTree.gen.ts`) is auto-generated by the Vite plugin during build/dev
- Frontend hooks follow the pattern in `src/routes/settings/-hooks/useSelectedRepositories.ts`
- CSS uses `@scope` for component isolation
